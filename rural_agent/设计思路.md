# 设计思路

报告严格按照以下六个章节顺序编制，后面的章节依赖于前面章节的研究内容。首先要研究政策，归纳出有利的政策。然后研究乡村现状，归纳可能发展的方向。然后进行乡村发展定位，指定发展大方向。然后进行产业项目细化分析，然后产业布局分析，最后对方案进行评价。

六个子章节分别是：
- 规划编制背景分析
- 村庄现状分析
- 产业发展方向分析
- 产业项目细化分析
- 产业项目布局分析
- 方案评价

设计一个RuralPlanOrchestrator管理类。负责章节的顺序执行和上下文传递。调用每一个子章节生成的智能体进行工作，然后收集结果并传递给后续智能体。该类需要存储各个章节的输出以及收集到的信息。需要按照指定顺序调用子图。

每个子章节设计一个针对该章节生成的智能体，智能体内部设计一个迭代的研究-审查循环，由定制的Researcher和Reviewer组成。Researcher首先负责收集信息并撰写初稿，然后交由Reviewer审查，Reviewer决定是否通过或者返回修改。Researcher根据返回的修改意见再次进行信息收集并修改初稿。

researcher需要集成底层的数据收集功能，包含网络信息收集接口以及ragflow数据收集接口。同时集成修改撰写初稿、修改初稿的接口。

reviewer则需要集成定制化的审查逻辑，接受初稿输入然后进行审查，返回修改意见或者通过。由于审查内容比较多，可以考虑内部构建多个审查方法，每个方法只针对一个方面进行审查。


经过研究，从网络爬取内容的scraper和retrievers封装相对耦合，依赖于GPTResearcher。现在我们需要构建自己的scraper和retrievers封装类。来满足数据获取的需求。
